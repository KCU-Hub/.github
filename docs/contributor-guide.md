# 학우용 기여 가이드

KCU-Hub는 공개 가능한 학습 기록을 다음 학우가 다시 쓸 수 있게 남기는 곳입니다. GitHub가 익숙하지 않아도 읽기, 제보하기, 템플릿 채우기부터 참여하실 수 있습니다.

## 3분 시작

| 하고 싶은 일 | 가장 쉬운 방법 |
|---|---|
| 자료를 찾고 싶습니다. | org 첫 화면이나 Pages에서 관심 있는 repo를 엽니다. |
| 링크가 깨졌습니다. | 해당 repo의 Issue에서 `문서 수정`을 선택해 알려주십니다. |
| 과목 정리를 공유하고 싶습니다. | 공개 가능한 요약과 참고 링크만 Markdown으로 남깁니다. |
| 스터디를 시작하고 싶습니다. | 목표, 기간, 회차별 기록 방식을 먼저 정합니다. |
| 프로젝트를 이어받고 싶습니다. | README의 실행 방법과 열린 Issue를 확인합니다. |

## GitHub를 몰라도 가능한 참여

| 단계 | 설명 |
|---|---|
| 읽기 | README와 Pages는 로그인하지 않아도 볼 수 있습니다. |
| 제보하기 | GitHub 계정이 있으면 Issue로 깨진 링크나 오타를 알려주실 수 있습니다. |
| 초안 남기기 | Markdown 문서 하나로 과목 요약, 스터디 회고, 프로젝트 기록을 시작할 수 있습니다. |
| PR 보내기 | 문서 수정에 익숙해지면 Pull Request로 직접 고칠 수 있습니다. |

## 좋은 기여 예시

- 과목을 다시 들을 학우가 이해할 수 있는 공개 요약
- 공식 문서, 공개 강의, 참고 자료 링크 모음
- 스터디 주차별 목표와 실제 진행 기록
- 프로젝트 실행 방법, 막힌 점, 다음에 할 일
- 오래된 링크나 틀린 설명을 고치는 작은 수정

## 올리면 안 되는 것

- 성적, 출석, 개인 상담 내용
- 이메일, 전화번호, 계정 식별자
- API 키, 접근 토큰, 비밀값, 비밀번호
- 비공개 강의 자료 원문
- 외부 인물 실명이나 민감한 내부 정보
- 아직 합의되지 않은 공식 입장, KPI, 운영 방침

## 과목 기록 기본 틀

```md
# 과목명

## 한 줄 요약

이 과목에서 무엇을 배우는지 적습니다.

## 다시 볼 개념

- 개념 1:
- 개념 2:
- 개념 3:

## 참고 링크

- 공개 자료 이름: URL

## 다음 학우에게

막혔던 지점, 먼저 보면 좋은 자료, 추천 순서를 적습니다.
```

## 스터디 기록 기본 틀

```md
# 스터디 이름

## 목표

무엇을 끝내면 성공인지 적습니다.

## 진행 방식

- 기간:
- 주기:
- 산출물:

## 회차 기록

| 회차 | 주제 | 남긴 자료 | 다음 할 일 |
|---|---|---|---|
| 1 |  |  |  |
```

## 프로젝트 기록 기본 틀

```md
# 프로젝트 이름

## 무엇을 해결하나요

프로젝트가 풀려는 문제를 적습니다.

## 실행 방법

필요한 설치와 실행 명령을 적습니다.

## 현재 상태

되는 것, 안 되는 것, 다음 작업을 구분합니다.

## 이어받을 때 먼저 볼 것

- 파일 또는 문서:
- 열린 Issue:
```

## 운영자에게 요청하기

새 repo가 필요하면 바로 만들기보다 먼저 Issue로 제안합니다. 첫 3개 산출물, 유지보수 방식, 공개 가능 여부가 정리되어 있으면 검토가 빨라집니다.
