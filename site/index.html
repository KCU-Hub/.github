비공식 학우 운영 허브

KCU-Hub는 공부 기록을 다음 학우에게 넘기는 곳입니다.

실시간 대화는 외부 채널에서 하고, 다시 찾아볼 자료와 프로젝트 기록은 GitHub에 남깁니다. 기본 언어는 한국어이며, GitHub가 익숙하지 않아도 읽기와 제보부터 시작하실 수 있습니다.

3분 시작 Markdown 가이드

처음 오셨다면

읽기만 해도 참여입니다.

1

자료 찾기

관심 있는 과목, 스터디, 프로젝트 repo의 README부터 확인합니다.

2

틀린 곳 제보

깨진 링크, 오래된 설명, 오타는 Issue로 알려주시면 됩니다.

3

기록 남기기

공개 가능한 요약, 참고 링크, 진행 회고를 Markdown으로 남깁니다.

기여 방법

GitHub 숙련도에 맞춰 참여하세요.

수준 할 수 있는 일 추천 행동
처음 읽기, 링크 저장, 오류 찾기 Issue로 오타와 깨진 링크를 알려주십시오.
익숙함 과목 요약, 스터디 기록 작성 Markdown 문서로 공개 가능한 내용을 남기십시오.
운영 참여 repo 제안, PR review, 자료 정리 첫 3개 산출물과 유지보수 방식을 함께 제안하십시오.

무엇을 남기나요

다음 학우가 다시 쓸 수 있는 자료를 남깁니다.

과목 기록

공개 가능한 개념 요약, 참고 링크, 복습 순서를 정리합니다.

스터디

목표, 회차별 자료, 진행 결과, 다음 할 일을 남깁니다.

프로젝트

실행 방법, 현재 상태, 열린 과제, 회고를 이어받기 쉽게 정리합니다.

운영 제안

새 repo가 필요한 이유, 첫 산출물, 유지보수 계획을 Issue로 제안합니다.

공개 기준

공개 repo에는 안전한 정보만 남깁니다.

남겨도 좋은 것

  • 직접 작성한 공개 요약
  • 공개 문서와 공개 자료 링크
  • 스터디 계획과 회고
  • 프로젝트 실행 방법과 TODO

남기면 안 되는 것

  • 성적, 출석, 상담 내용
  • 이메일, 계정 식별자, 연락처
  • API 키, 접근 토큰, 비밀값, 비밀번호
  • 비공개 강의 자료 원문

작게 시작해도 충분합니다.

오타 하나, 링크 하나, 짧은 회고 하나가 다음 학우에게는 길잡이가 됩니다.

Issue로 제안하기 KCU-Hub repo 보기
